Good question. JavaScript evolved step-by-step to handle asynchronous operations more cleanly. The order is:

1️⃣ Callbacks → 2️⃣ Promises (.then) → 3️⃣ Async/Await

Let’s understand why each came.

1️⃣ Callbacks (First approach)

Earlier, async operations used callbacks.

Example:

setTimeout(function () {
  console.log("Order created");

  setTimeout(function () {
    console.log("Payment done");

    setTimeout(function () {
      console.log("Order shipped");
    }, 1000);

  }, 1000);

}, 1000);

Problem ❌
This creates Callback Hell (Pyramid of Doom).

Issues:

Hard to read

Hard to debug

Hard to manage errors

Nested code grows quickly

2️⃣ Promises (.then) — Solution to Callback Hell

To solve callback hell, Promises were introduced.

Example:

createOrder(cart)
.then(function(orderId) {
    return proceedToPayment(orderId);
})
.then(function(paymentInfo) {
    return showOrderSummary(paymentInfo);
})
.then(function(summary) {
    return updateWallet(summary);
})
.catch(function(err) {
    console.log(err);
});

Advantages ✅

No deep nesting

Better error handling with .catch()

Promise represents a future value

A Promise has 3 states

Pending
Resolved (fulfilled)
Rejected

But still there was a small issue.

Even though it's better than callbacks, long .then chains are still harder to read.

3️⃣ Async / Await — Cleaner syntax over Promises

async/await was introduced in ES2017.

Important point:

⚡ Async/Await is just syntactic sugar over Promises.
Internally it still uses Promises.

Example:

async function orderFlow() {
  try {
    const orderId = await createOrder(cart);
    const paymentInfo = await proceedToPayment(orderId);
    const summary = await showOrderSummary(paymentInfo);
    const wallet = await updateWallet(summary);
  } catch (err) {
    console.log(err);
  }
}

Advantages ✅

Looks like synchronous code

Much easier to read

Easier debugging

No long .then() chains

4️⃣ Timeline Summary
Approach	Problem Solved	Issue
Callbacks	Basic async handling	Callback hell
Promises (.then)	Removes nested callbacks	Long chains
Async/Await	Cleaner syntax for promises	Still promise internally

Flow evolution:

Callbacks
   ↓
Promises (.then / .catch)
   ↓
Async / Await
