Good question. I’ll explain pin-to-pin (step by step) what happens from fetch() → response → readable stream → .json() → final object.

We’ll use your code.

<script>
async function handlepromise(){
  const API = "https://api.github.com/users/mosurupodiadarshreddy";

  const data = await fetch(API);
  const result = await data.json();

  console.log(result);
  console.log(result.id);
};

handlepromise();
</script>
1️⃣ fetch() – What it is

fetch() is not part of JavaScript itself.
It is provided by the browser Web API.

It is used to make HTTP requests (call APIs).

Example request sent to server:

GET https://api.github.com/users/mosurupodiadarshreddy

Server responds with user information in JSON.

2️⃣ What fetch() returns

fetch() immediately returns a Promise.

const data = await fetch(API);

Equivalent to:

fetch(API).then(response => ...)

So internally:

fetch()
↓
returns Promise
↓
Promise resolves with Response object
3️⃣ Response Object

When the request completes, the promise resolves and gives a Response object.

Example structure:

Response {
  status: 200
  ok: true
  headers: ...
  body: ReadableStream
}

Important property:

body → ReadableStream

This means the response is not plain JSON yet.

4️⃣ Why the body is a ReadableStream

When a server sends data, it does not always send everything at once.

It sends data in chunks.

Example idea:

chunk1
chunk2
chunk3

This streaming system is called:

Readable Stream

Benefits:

Large files can be processed gradually

Faster performance

Memory efficient

5️⃣ Converting stream to JSON

Because the body is a stream, we must read and convert it.

That is what .json() does.

const result = await data.json();

Important:

data.json() returns another Promise

Why?

Because it must:

read the stream
↓
combine chunks
↓
parse JSON

Full pipeline:

First, your code:

async function handlepromise(){

1️⃣ What try...catch Does

try...catch is used to handle runtime errors.

Structure:

try {
   // code that might fail
}
catch(err) {
   // handle the error
}

If an error happens in try, control moves to catch.

2️⃣ Power of try...catch with async/await

Before async/await, errors in promises were handled like this:

fetch(API)
.then(res => res.json())
.then(data => console.log(data))
.catch(err => console.log(err));

With async/await we use try/catch, which looks like normal synchronous error handling.

try {
   const data = await fetch(API);
} catch(err) {
   console.log(err);
}

This is cleaner and easier to read.

3️⃣ Step-by-Step Execution of Your Code
Step 1 — handlepromise() runs

Call stack:

handlepromise()
Step 2 — Inside try block
const data = await fetch(API);

fetch() sends a network request to GitHub.

Async function pauses at await.

Step 3 — Server responds

GitHub returns:

404 Not Found

But here is a very important rule:

fetch() does NOT reject the promise for HTTP errors like 404

It only rejects for network failures.

So the promise still resolves.

data becomes a Response object:

Response {
  status: 404,
  ok: false,
  body: ReadableStream
}
4️⃣ Next Line Executes
const result = await data.json();

GitHub returns a JSON error response like:

{
  "message": "Not Found",
  "documentation_url": "https://docs.github.com/rest"
}

So result becomes:

{
  message: "Not Found"
}
5️⃣ Console Output

Your console prints:

{ message: "Not Found" }

Your catch block does NOT run.

Why?

Because no JavaScript error occurred.

6️⃣ When catch Actually Runs

catch runs only if a real error happens, like:

❌ Network failure

Example:

fetch("https://wrongdomain1234.com")

Now fetch rejects the promise.

Then catch runs:

Failed to fetch
❌ JSON parsing failure

If .json() fails, catch also runs.
