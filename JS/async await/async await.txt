ASYNC   :::: is a keyword used to create async function it always returns promise


1️⃣ Perfect Meaning of async
async means the function will return a Promise and can handle asynchronous operations in a clean way.
Simple definition:
async = a function that works with promises and allows using await

Example:

async function getData() {
  return "hello";
}

Internally this becomes:

function getData() {
  return Promise.resolve("hello");
}

So rule:

normal function → returns value
async function → returns Promise

===========================================================================================
2️⃣ What await does inside async
 
await tells JavaScript:

pause this async function
until the promise is resolved

Example:

async function getUser() {
  const data = await fetch("/api/user");
  const user = await data.json();
  console.log(user);
}

Without await you would write:

fetch("/api/user")
.then(res => res.json())
.then(user => console.log(user));

So async/await is cleaner syntax over promises.

3️⃣ Why async/await was introduced

Before async/await we had Promise chains:

.then()
.then()
.then()
.catch()

Which becomes messy.

Example:

createOrder()
.then(order => payment(order))
.then(payment => ship(payment))
.then(data => console.log(data))
.catch(err => console.log(err));

Async/await makes it look like synchronous code:

async function process() {
  try {
    const order = await createOrder();
    const payment = await payment(order);
    const ship = await ship(payment);
  } catch(err) {
    console.log(err);
  }
}

Much easier to read.

4️⃣ Where we use async in real projects

You use async whenever you deal with operations that take time.

Examples:

API calls
async function loadUsers() {
  const res = await fetch("/api/users");
  const users = await res.json();
  setUsers(users);
}

Very common in React apps.

Multiple APIs

Example dashboard loading data:

async function loadDashboard() {
  const users = await getUsers();
  const orders = await getOrders();
  const revenue = await getRevenue();
}


5️⃣ When NOT to use async

You do not need async for synchronous code.

Example:

❌ unnecessary:

async function add(a,b){
  return a+b;
}

Better:

function add(a,b){
  return a+b;
}

6️⃣ Simple Rule (Best to Remember)

Use async when dealing with:

API calls
Database
File system
Timers
Network requests

Basically anything that takes time.

Short answer:

The remaining code after await is not placed into the microtask queue immediately.

The Rule

When JavaScript encounters:

await p1

pause the async function
↓

The .then callback cannot enter the microtask queue

7️⃣ Summary (very important)

| Situation        | fetch behavior   |
| ---------------- | ---------------- |
| 200 OK           | Promise resolves |
| 404 Not Found    | Promise resolves |
| 500 Server Error | Promise resolves |
| Network failure  | Promise rejects  |

So 404 is not treated as an error by fetch.

8️⃣ The correct production pattern

Real applications usually do:

  try{
    const res = await fetch(API);

      throw new Error(`HTTP Error ${res.status}`);
    }

    const data = await res.json();
    return data;

  }catch(err){
    console.error(err);
  }
}
