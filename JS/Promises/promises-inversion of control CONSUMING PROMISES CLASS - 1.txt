🔐 PROMISES IN JAVASCRIPT — THE FINAL TRUTH
1️⃣ Where Promises DO come from ✅

A Promise is created ONLY in these situations:

✅ Case 1: Function marked async
async function greet() {
  return "Hello";
}

const result = greet();

🔍 What happens:
greet() returns a Promise
Promise resolves to "Hello"
result instanceof Promise; // true

📌 Rule

Every async function ALWAYS returns a Promise
(even if you return a normal value)

✅ Case 2: Explicit new Promise()
function createOrder(cart) {
  return new Promise((resolve, reject) => {
    // async work
    resolve("orderId123");
  });
}

const p = createOrder(cart);

✔️ p is a Promise

📌 Rule

If you see new Promise, a Promise is being returned

✅ Case 3: Built-in async APIs
fetch("/api/data");
Returns a Promise automatically.
Common examples:
| API                      | Returns         |
| ------------------------ | --------------- |
| `fetch()`                | Promise         |
| `axios.get()`            | Promise         |
| `setTimeout()`           | ❌ NOT a Promise |
| `fs.promises.readFile()` | Promise         |

📌 Rule

If an API is async and chainable with .then(), it returns a Promise

✅ Case 4: .then() ALWAYS returns a Promise
fetch(url)
  .then(res => res.json())
  .then(data => console.log(data));

Each .then() returns a new Promise.

📌 Rule

Promise chaining works because .then() returns a Promise

2️⃣ Where Promises DO NOT come from ❌
❌ Function declaration or assignment
const data = function greet() {
  return 10;
};

data → function

❌ Not a Promise

❌ Calling a normal (non-async) function
function add(a, b) {
  return a + b;
}

const result = add(2, 3);

result → number

❌ Not a Promise

❌ Arrow functions (by default)
const fn = () => "hello";

❌ Not a Promise
(unless marked async)

❌ Callbacks
setTimeout(() => {
  console.log("Hi");
}, 1000);

Uses callback

❌ No Promise returned

3️⃣ The BIGGEST CONFUSION (MUST REMEMBER)
❌ WRONG thinking

“This function is a promise”

✅ CORRECT thinking

“This function RETURNS a promise WHEN executed”

const fn = async () => "ok";

fn;    // function
fn();  // Promise
4️⃣ One-screen cheat table 🧠
| Code                   | Is Promise?      |
| ---------------------- | ---------------- |
| `function f(){}`       | ❌                |
| `f`                    | ❌                |
| `f()`                  | ❌ (unless async) |
| `async function f(){}` | ❌                |
| `f()` (async)          | ✅                |
| `new Promise()`        | ✅                |
| `fetch()`              | ✅                |
| `.then()`              | ✅                |

5️⃣ How to IDENTIFY a Promise in 5 seconds ⚡

Ask these 3 questions:

Is the function marked async?
Does it return new Promise()?
Is it a known async API like fetch, axios, DB calls?

👉 If YES to any, then:

value instanceof Promise === true
6️⃣ Interview-ready final answer 🎯

Promises are returned only by async functions, functions that explicitly return new Promise, or built-in asynchronous APIs like fetch.
A function itself is never a Promise — only its execution may return one.


✅ When does a function RETURN a Promise?

A function returns a Promise ONLY in these situations 👇

1️⃣ When it explicitly returns new Promise(...)
function getData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve("data received");
    }, 1000);
  });
}

const result = getData();

✔️ result is a Promise
📌 Rule
If you see return new Promise(...), the function returns a Promise.

2️⃣ When it returns another Promise
function getUser() {
  return fetch("/api/user"); // fetch already returns a Promise
}

const result = getUser();
✔️ result is a Promise
📌 Rule
Returning a Promise → function returns a Promise.

3️⃣ When the function is marked async (for later learning)
async function greet() {
  return "hello";
}

const result = greet();

✔️ result is a Promise
📌 Rule
async forces return value to be wrapped in a Promise.
(Just remember this for now; no need to use it yet.)

4️⃣ When .then() is used in the return chain
function getData() {
  return fetch("/api/data").then(res => res.json());
}

const result = getData();

✔️ result is a Promise
📌 Rule
.then() ALWAYS returns a Promise.

❌ When a function does NOT return a Promise
❌ Normal return values
function add(a, b) {
  return a + b;
}

❌ Not a Promise
❌ Async work inside but no Promise returned
function greet() {
  setTimeout(() => {
    console.log("hello");
  }, 1000);
}

const result = greet();
❌ result is undefined, NOT a Promise
📌 Important

Using async code inside a function does NOT make the function return a Promise automatically.

🧠 One-look identification checklist

Ask these 4 questions:

1️⃣ Does the function return new Promise()?
2️⃣ Does it return fetch / axios / db call?
3️⃣ Does it return something with .then()?
4️⃣ Is it marked async?

👉 If YES to any → returns Promise
👉 If NO to all → NOT a Promise
🔒 Final one-line truth (memorize)

A function returns a Promise only when it explicitly returns a Promise or is declared async.
Async operations inside a function do not automatically make it return a Promise.
